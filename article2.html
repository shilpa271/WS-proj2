AAA Testing Tutorial

Introduction to AAA Patterns for Testing

So, we've successfully finished coding. We finally have, for all intents and purposes, a fully functional program. Now what? An inexperienced programmer would call it a done day, and move on to another project. This may be fine with a smaller project, but this would eventually cause problems later on in development, especially for complex business projects with many requirements. Now, pose this question to an experienced programmer who has been in the field for many years, and they would all say without hesitation that testing is the next step. Picture this, you send this project to friends and family to show them your hard work. Within a few hours of sending it, you get a call from your younger brother saying that the program broke after using it for an hour. This is exactly why testing is so important; users of the program will most likely break the program extremely quickly. Through testing, we stress test the program so that when it is finally sent to the public, the programs' integrity remains strong.

Why Testing is Important

  • How else would you know if your code works?
  • Proper testing can detect any bugs and defects in the code
  • It can also find security flaws
  • It ensures the code does exactly what it is supposed to do before publishing
  • Lastly, it will prove there will not be any unexpected results
